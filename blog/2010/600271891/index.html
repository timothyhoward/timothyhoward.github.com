What is Open?

Yesterday’s post on Flash’s claims of openness was linked to on Daring Fireball (and later followed up) and resulted in a lot of emails telling me how wrong I am.

Admittedly, I was wrong and posted an amendment. In Gruber’s followup he wrote:

I still say Whimpey’s conclusion stands — it’s not the tools that make Flash a closed platform, it’s that the runtime, Flash Player, is completely controlled and defined by Adobe. Adobe’s argument is that Flash is an open part of the web but that there just happens to exist only one implementation, their own closed-source Flash Player.

Adobe has published the SWF file format specification which theoretically allows anyone to build authoring tools and interpreters for Flash content. In practice the number one open source Flash player (or at least the only one with any presence online) barely supports anything. From the gnash website:

[…] and supports most SWF v7 features and some SWF v8 and v9

SWF is currently at version 10. The page proudly listing open source and free development tools has a section on players:

While the specs for creating SWF files are freely available, it’s not currently possible to create fully featured Flash Players as Open Source. The main reason is that Adobe’s implementation of the Flash Player makes use of technology licensed from several companies.

So my original point stands:

Adobe: not open, claim to be.

Apple: not open, don’t claim to be, contribute heavily to that which is truly open.
